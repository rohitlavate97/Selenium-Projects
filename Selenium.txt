============================================================
AUTOMATION TESTING FRAMEWORK
============================================================


FRAMEWORK SUMMARY


This document describes a complete, scalable, production-ready
Selenium WebDriver automation framework created using Java,
Maven, TestNG, and Selenium.

The framework is designed to be:
- Easy to understand
- Easy to extend
- Easy to maintain
- Suitable for real-time projects
- Suitable for interview explanation
- Suitable for enterprise-level automation


------------------------------------------------------------
WHAT HAS BEEN CREATED
------------------------------------------------------------

A complete Selenium WebDriver automation framework with:

- Clean layered architecture
- Centralized configuration management
- Thread-safe WebDriver handling
- Support for multiple browsers
- Support for multiple environments
- Data-driven testing support
- Parallel execution support
- Detailed reporting
- Logging and screenshot capture
- Ready-to-use sample implementation
- Execution via command line, scripts, and IDE


------------------------------------------------------------
CORE COMPONENTS
------------------------------------------------------------

Total files created in framework: 27+


------------------------------------------------------------
1. CONFIGURATION LAYER
------------------------------------------------------------

This layer is responsible for managing all configuration related
to the framework and test execution.


1.1 ConfigReader.java

Purpose:
- Reads configuration values from property files
- Provides a centralized way to access configuration values
- Avoids hardcoding values in code

Responsibilities:
- Load config.properties file
- Provide getter methods for browser, environment, url,
  credentials, execution flags, and timeouts


1.2 config.properties

Purpose:
- Stores environment and execution related configuration

Contains values such as:
- browser = chrome / firefox / edge
- environment = dev / qa / staging / prod
- app.url
- username
- password
- headless
- maximize
- incognito
- implicit.wait
- explicit.wait
- page.load.timeout
- screenshot.on.failure
- screenshot.on.pass
- report.name
- report.title
- thread.count


1.3 log4j2.xml

Purpose:
- Logging configuration for the framework

Features:
- Console logging
- File logging
- Daily rolling logs
- Supports DEBUG, INFO, WARN, ERROR


------------------------------------------------------------
2. WEBDRIVER MANAGEMENT
------------------------------------------------------------

2.1 DriverManager.java

Purpose:
- Create and manage WebDriver instances
- Ensure thread-safe execution

Features:
- ThreadLocal<WebDriver>
- Supports Chrome, Firefox, Edge
- Uses WebDriverManager
- getDriver() and quitDriver() methods
